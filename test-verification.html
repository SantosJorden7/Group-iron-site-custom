4. Integration Testing Instructions

To fully test the Collection Log feature in a production environment:

  1. Deploy the application using instructions in DEPLOY.md
  2. Navigate to the Collection Log page at /collection-log
  3. Verify the Sources Panel shows status for all three data sources
  4. Click the manual sync button to trigger a data refresh
  5. Observe that items show appropriate source badges
  6. Verify that tooltips appear when hovering over Wiki-enriched items
  7. Test with the RuneLite plugin to confirm plugin data takes priority

Expected Result: The Collection Log should show items from all three data sources, with plugin data taking priority, followed by Wise Old Man, then Wiki data. Each source should be clearly indicated with a badge.
